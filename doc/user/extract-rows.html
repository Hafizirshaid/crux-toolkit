extract-rows

Usage:
crux extract-rows [options]  <tsv filename>  <column name> <column value>
Description:

Given a tab-delimited filename, a column name and a column cell value, extract the rows that have the matching values for that column.

Input:
  • <tsv filename> – A tab-delimited file with column headers.
  • <column name> – A column name
  • <column value> – A cell value for a column
Output:
    The program prints to standard output the rows for which the value observed in the specified column of the tab-delimited input file match the <column value> given on the command line.
Options:
  • --header T|F – Print the header line of the file, in addition to the rows that match. Default=T.
  • --comparison <operator> – Specify the operator that is used to compare an entry in the specified column to the value given on the command line. Legal values are as folows:
    • string-equal – Compare the two values as strings.
    • numeric-equal – Compare the two values as numbers.
    • gt – The is value in the file greater than the value specified on the command line?
    • gte – The is value in the file greater than or equal to the value specified on the command line?
    • lt – The is value in the file less than the value specified on the command line?
    • lte – The is value in the file less than or equal to the value specified on the command line?
    The default value is string-equal.
  • --verbosity <0-100> – Specify the verbosity of the current processes. Each level prints the following messages, including all those at lower verbosity levels: 0-fatal errors, 10-non-fatal errors, 20-warnings, 30-information on the progress of execution, 40-more progress information, 50-debug info, 60-detailed debug info. Default = 30.
